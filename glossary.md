# 用語集

ClaudeCodeを使う中で出てきた言葉の説明。
`/ask <用語>` で追加される。

---

## claude -p
category: スクリプト

CCがスクリプト（自動処理）の中に書く、Claudeへの一回きりの質問命令。

**【CCがこれを使うとき、何が起きているか】**
CCの回答にこれが出てきたとき、それは「自動で動くプログラム」を書いている場面です。`claude -p "〇〇を教えて"` という形で、プログラムが自動的にClaudeに質問して、返ってきた答えを受け取ります。あなたが画面を見ていなくても勝手に動きます。

**【挙動イメージ】**
上司が部下に「これ調べといて、結果だけ紙に書いて机に置いといて」と頼むようなイメージ。部下（Claude）は答えだけ返してそれで終わり。会話は続かない。

**【実例】**
- 記事のキーワードリストが100件あって、それを自動でカテゴリ分けするスクリプトを作るとき、CCが1件ずつ `claude -p "このKWは商標系？情報系？"` と投げて判定結果を受け取る
- CCがSlackに通知を送るスクリプトを書くとき、送る文章の日本語チェックを `claude -p "この文章おかしい箇所ある？"` に任せる

**【あなたが知っておくべきこと】**
- あなたが自分で入力するものではない。CCが自動処理の中に組み込むもの
- これが出てきたら「裏でCCが別のClaudeに質問している」と思えばOK

## config
category: CC設定

CCの動き方を決める「設定ファイル」のこと。

**【CCがこれを使うとき、何が起きているか】**
CCの回答に「config」が出てきたとき、それはCCの動き方を制御するファイル（`~/.claude/settings.json`）を読み書きしている場面です。言語設定・使うモデル・許可する操作の範囲など、CCの基本的なふるまいがここで決まります。

**【挙動イメージ】**
スマホの「設定」アプリと同じ。通知をONにしたり、言語を変えたりする場所。CCにとっての「設定アプリ」がconfigファイルです。

**【実例】**
- CCが「日本語で回答する」「sonnetモデルを使う」という設定で動いているのは、configに `"language": "日本語"` と書かれているから
- CCが新しい操作を許可するとき、configの `permissions` という項目を書き換えている

**【あなたが知っておくべきこと】**
- 場所は `~/.claude/settings.json`。直接開いて見ることもできる
- CCが「設定を変更します」と言ったときは、このファイルが書き換わる

## Dispatch
category: ツール

「今すぐ動け」という合図を別のシステムに送って処理を起動させる仕組み。

**【CCがこれを使うとき、何が起きているか】**
CCの回答に「Dispatch」が出てきたとき、それは「別のシステムやプログラムに対して開始の信号を送る」処理を書いている場面です。受け取った側は信号を受けて動き始めます。CCが何かを自動実行させたいとき、その「起動命令」がDispatchです。

**【挙動イメージ】**
工場の司令塔が各部署に「今すぐ出荷作業を開始しろ」と無線で指令を出す感じ。指令（Dispatch）を受けた部署（システム）が一斉に動き始める。

**【実例】**
- CCがGitHub Actionsの自動処理を手動で起動するとき、`workflow_dispatch` という信号を送る。これがDispatch
- 用語集のページを更新したとき「GitHub Pagesのビルドをdispatchした」という表現で出てくることがある

**【あなたが知っておくべきこと】**
- Dispatchは「起動命令を送った」という意味。実際の処理は受け取った側がやる

## -f
category: コマンドオプション

コマンドの後ろにつける「追加の指示」のひとつ。「強制的にやれ」「ファイルを指定する」など、コマンドによって意味が変わる。

**【CCがこれを使うとき、何が起きているか】**
CCの回答に `-f` が出てきたとき、それはコマンド（命令文）に「オプション」をつけている場面です。`-f` はコマンドによって意味が違います。たとえば `rm -f` なら「確認なしで強制削除」、`git push -f` なら「強制的に上書き送信」、`docker build -f` なら「使う設計図ファイルを指定」という意味です。`-` のあとにアルファベット1文字がつく形は「フラグ」や「オプション」と呼ばれ、コマンドの動きを細かく調整するものです。

**【挙動イメージ】**
料理の注文に「辛さ増しで」「大盛りで」とオプションをつけるのと同じ。`-f` は注文票に書き加える追加指示のひとつです。何の注文（コマンド）につけるかで意味が変わります。

**【実例】**
- CCが `git push -f origin main` と書いたとき → 「リモートの内容を無視して、こちらの内容で強制的に上書きする」という意味。これは他の人の変更を消す可能性がある危険な操作
- CCが `rm -f ファイル名` と書いたとき → 「本当に消していいですか？」の確認を飛ばして即座に削除するという意味
- CCが `docker build -f Dockerfile.prod .` と書いたとき → 「Dockerfile.prodという名前の設計図ファイルを使ってビルドしろ」という意味

**【あなたが知っておくべきこと】**
- `-f` は「force（強制）」の略であることが多い。強制＝確認を飛ばすということなので、データが消えるリスクがある
- CCが `-f` つきの削除コマンドやGit操作を提案してきたら、「本当にそれでいいか」を自分で判断してからOKを出すこと
- `-f` が「file（ファイル指定）」の略になるコマンドもある。文脈で判断する
